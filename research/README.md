This folder is the intake and run workspace for the cyclic methodology.

1. Gather primary sources from model vendors, benchmark authors, and research groups.
2. Extract durable claims about capabilities, limitations, and operating constraints.
3. Separate descriptive claims (what models do) from normative claims (how we should use them).
4. Turn repeated patterns into status-labeled postulates.
5. Feed validation outcomes back into the next cycle.

- `questions.md`: question backlog with cycle status tracking.
- `sources.md`: curated primary sources and extraction notes.
- `cycles/`: per-cycle records (inputs, outputs, deltas, next-cycle queue).
- Capability: what generative models reliably do well.
- Limitation: where they fail, drift, hallucinate, or need verification.
- Parameter: what governs behavior in practice.
- Orchestration: how multiple agents and humans should coordinate.
- Domain application: how postulates adapt when applied to a specific domain scenario.
